# 需求到任务

[English](README.md) | 简体中文

当一个需求已经真实，但还模糊到工程师或 AI assistant 不能安全开工，就看这个场景。

## 什么时候看

- 一个请求可以被理解成好几种做法。
- 需要先对"完成"达成一致。
- 工作可能牵涉产品行为、设计、数据或发布。
- 你希望代码开始前，先有一个小而可 review 的任务。

## 常见路径

- 把松散想法变成简短 spec。
- 把 PRD 或 issue 变成实现计划。
- 把小需求变成 AI-ready task。
- 写代码前检查设计流程。
- 改既有系统前，先规划 brownfield 影响。

## 第一步

先写一张短任务卡：

- 要改什么？
- 哪些东西不要改？
- 怎么知道已经完成？
- Merge 前需要检查什么？
