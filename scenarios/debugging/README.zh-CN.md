# 调试

[English](README.md) | 简体中文

当你有症状，但还没有被证明的原因，就看这个场景。

## 什么时候看

- 你有日志、trace、截图或失败测试。
- 故障跨过了前端、后端、基础设施或数据边界。
- AI 总是在证明哪里坏了之前就开始提修复方案。
- 团队需要证据，再决定怎么改代码。

## 常见路径

- 失败测试调查。
- 日志和 trace 审查。
- 最小复现。
- 回归 bisect。
- 生产问题 triage。

## 第一步

改代码前，先写下证据、当前假设和下一步要检查什么。
