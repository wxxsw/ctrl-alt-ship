# 项目上下文记忆

[English](README.md) | 简体中文

当 AI 总是漏掉项目怎么工作，就看这个场景。

## 什么时候看

- 每个新会话都要重复解释 setup。
- AI 漏掉本地风格、架构或 review 预期。
- 项目知识散在聊天、工单、文档和人的脑子里。
- 新贡献者需要更快理解 repo。

## 常见路径

- 为 AI-assisted work 写一份短项目指南。
- 做一张轻量 repo map。
- 把反复出现的决策放进长期项目文档。
- 个人研究笔记先单独放，准备好后再进入团队文档。

## 第一步

先写下一个新 AI 会话最需要知道的基础信息：

- 项目怎么运行和测试。
- 哪些目录重要。
- 本地约定是什么。
- 哪些区域需要格外小心。

## 相关入口

- 如果任务本身不清楚，见 [需求到任务](../requirements-to-tasks/README.zh-CN.md)。
- 如果决策也需要给人长期保存，见 [文档与知识](../documentation-knowledge/README.zh-CN.md)。
