# Project Context Memory

[English](README.md) | [简体中文](README.zh-CN.md)

Use this when AI keeps missing how your project works.

## Use it when

- Every new session needs the same setup explanation.
- AI misses local style, architecture, or review expectations.
- Project knowledge is scattered across chats, tickets, docs, and people's heads.
- New contributors need a quicker way to understand the repo.

## Common paths

- Add a short project guide for AI-assisted work.
- Create a lightweight repo map.
- Move repeated decisions into durable project docs.
- Keep personal research notes separate until they are ready for the team.

## First move

Write down the basics a new AI session needs:

- How to run and test the project.
- Which directories matter.
- What local conventions should be followed.
- Which areas need extra care.
