# 文档与知识

[English](README.md) | 简体中文

当决策和项目知识总是消失在聊天里，就看这个场景。

## 什么时候看

- 架构或产品决策之后还会被用到。
- 新队友缺少项目背景。
- 文档过期、分散，或者更新成本太高。
- 最近一次工作里的经验应该变成可复用知识。

## 常见路径

- 决策记录。
- 架构笔记。
- Runbooks。
- Onboarding docs。
- 个人知识系统。

## 第一步

一次有意义改动后，写下决策、取舍和未来维护提示。

## 相关入口

- 如果缺失的是 AI 工作上下文，见 [项目上下文记忆](../project-context-memory/README.zh-CN.md)。
