# Initial Roadmap

This roadmap captures the first version of the project direction.

## Phase 1: Make The Map Useful

Goal: make the stable scenario map clear enough that a reader understands the project in one visit.

Priority work:

- Refine the 10 initial scenario pages.
- Add decision trees to the highest-value scenarios.
- Add stronger examples to Automated Verification, Requirements to Tasks, and Project Context Memory.
- Add first-class tool notes for the tools already mentioned in scenario pages.
- Create the first real weekly radar issue.

## Phase 2: Make Contributions Easy

Goal: make it easy for others to add field-tested workflows without breaking the structure.

Priority work:

- Expand contribution templates.
- Add issue labels and review checklist.
- Add example scenario branch contributions.
- Define source and evidence expectations.
- Add a "good first contribution" list.

## Phase 3: Make It Discoverable

Goal: help developers understand the project from GitHub search, social sharing, and README scanning.

Priority work:

- Improve README examples.
- Add GitHub topics.
- Add a compact scenario index.
- Add example user journeys by role.
- Add bilingual positioning where helpful.
