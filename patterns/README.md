# Patterns

Patterns are reusable workflow ideas that can appear in more than one scenario.

This section is intentionally light for now. The public map should help readers find the right situation first. Pattern details can be expanded when they are tied to a concrete scenario.

Early examples:

- Spec before code
- Smoke test before agent work
- Repo map before large refactor
- Review against plan
- Human approval for write-capable tools
- Trace before tuning
