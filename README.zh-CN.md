# Ctrl Alt Ship

[English](README.md) | 简体中文

AI 编程工具变化很快。真正难的，还是在一个具体工程场景里判断：我现在该试什么？

Ctrl Alt Ship 是一张场景地图。先看你眼前是什么工作，再打开对应场景，把它当成一份独立指南来用。

## 从你的处境开始

| 当前处境 | 从这里开始 |
| --- | --- |
| 需求还很模糊，需要变成能执行的任务 | [需求到任务](scenarios/requirements-to-tasks/README.zh-CN.md) |
| AI 总是漏掉项目上下文或本地约定 | [项目上下文记忆](scenarios/project-context-memory/README.zh-CN.md) |
| 想用 AI 做实现、迁移或重构 | [编码与重构](scenarios/coding-and-refactoring/README.zh-CN.md) |
| 有故障现象，但根因还没被证明 | [调试](scenarios/debugging/README.zh-CN.md) |
| AI 改了代码，你需要证明它仍然能跑 | [自动化验证](scenarios/automated-verification/README.zh-CN.md) |
| AI 生成的工作需要在 merge 前 review | [代码审查与质量门禁](scenarios/code-review-quality-gates/README.zh-CN.md) |
| 决策和项目知识总是散落在聊天里 | [文档与知识](scenarios/documentation-knowledge/README.zh-CN.md) |
| 一个改动需要 release notes、灰度或回滚思考 | [发布与变更管理](scenarios/release-change-management/README.zh-CN.md) |
| 日志、告警和聊天记录需要整理成事故时间线 | [事故响应](scenarios/incident-response/README.zh-CN.md) |
| 团队需要统一 AI 使用边界 | [团队 AI 治理](scenarios/team-ai-governance/README.zh-CN.md) |

## 怎么用

选择最贴近当前工作的场景。每个场景页都是一份独立深挖指南，包含判断方式、主流解决路径、工具取舍、实操流程、示例、验证清单和常见坑。

工具说明只用于比较。先确定场景，再看工具。
